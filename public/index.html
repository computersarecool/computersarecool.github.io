To conveniently use Latex for mathematics in web pages, a nice tool seems to be MathJax. (I found it supported by APS, the American Physical Society.) Here is the briefest, while sufficient, intro to it.

A one–line JavaScript call is needed in the preamble, which loads the program from their web site. Then, one simply types latex where math is wanted, along with HTML (for example \( e^{\boldsymbol{i} \, (\beta+\pi)} \) here), using latex's usual math–mode delimiters. The exponent above is entered, in the middle of HTML code,by 

    ... (for example \( e^{\bm{i} \, (\beta+\pi)} \) here) ...

With MathJax present this is recognized as a chunk of mathematics to process, like it would be in a latex document, and is handled as such. Then it is displayed by HTML-CSS or MathML (so that it scales well etc) — these are not images. One can right-click on an equation and adjust some properties, copy source, etc. To get a nice and quick start, see their documentation. Here is an example copied from that page:

When $a \ne 0$, there are two solutions to \( ax^2 + bx + c = 0 \) and they are $$ x = {-b \pm \sqrt{b^2-4ac} \over 2a}. $$

This example is produced by: 

When $a \ne 0$, there are two solutions to \( ax^2 + bx + c = 0 \) and they are
$$
x = {-b \pm \sqrt{b^2-4ac} \over 2a}.
$$

There are some things to watch out for, like (single) dollar signs, “<” signs, etc, since both HTML and Latex are processed, one after another (in this order). For this example (using $ for inline–math delimiters) one does need another JavaScript segment in the preamble, see below.

If you look at the HTML source of this page, you will find three short JavaScript entries in the preamble. The last one calls the program from MathJax site, and this apparently should be last. This is all that must be called, and one can do full math. (The program can be installed locally instead.)

The other JavaScript segments are for MathJax configuration. The first one allows use of dollar signs to delimit inline math, along with (default) \( ... \). The second one loads Tex extensions, here one that will add \boldsymbol{...} automatically around math that happens to be within bold HTML, and one that enables use of AMS symbols. Extensions can also be included directly in the script that calls MathJax, and this is how AMS–math is enabled in this page. See documentation, in particular for Tex/Latex support.

For nice examples, go to a MathJax demo page. (Right-click on equations, choose "Show source;" it's full latex.)
